# Как создать новый проект с Next.js

Next.js ([nextjs.org](https://nextjs.org)) — это библиотека, чтобы делать сайты и веб-приложения.

Здесь написана инструкция, как сделать проект на Next.js.

## Установка Visual Studio Code

Visual Studio Code — это такой клевый редактор.

<Figure image="/vsc.png" noBorder />

Гуглите "Visual Studio Code" и делаете по инструкции.

## Установка Node.js

Node.js — это штука, которая умеет запускать серверый код на JavaScript. То есть, не код, который рисует сайт, а код, который отвечает на запросы, общается с базой данных и так далее.

Открываете [nodejs.org](https://nodejs.org/) и устанавливаете. Лучше брать версию, у которой рядом буквы "LTS".

## Создание проекта

Создаете папку в любом месте. В этой папке будет проект.

Открываете Visual Studio Code. Там "Open folder" и выбираете эту папку.

Теперь открываете терминал в Visual Studio Code и набираете там:

```bash Terminal
npx create-next-app . --typescript
```

Ждете, пока оно все установит. Если почему-то прервалось — ту же команду заново.

## Запуск проекта

В терминале:

```bash Terminal
npm run dev
```

И открыть в браузере адрес [localhost:3000](http://localhost:3000).

## Деплой проекта

"Деплой" — это когда загружаешь проект на сервер где-то в интернете и он там работает (а не локально на компьютере).

Деплоить мы будем с помощью Vercel ([vercel.com](https://vercel.com)), потому что он бесплатный и хорошо работает. Но вообще есть тысяча разных других штук, которые можно использовать.

Сначала надо зарегистрироваться на [vercel.com](https://vercel.com). Затем добавить в проект команду `vercel`:

```bash Terminal
npm install -D vercel
```

Залогиниться в `vercel` из проекта (нужно только один раз):

```bash Terminal
npx vercel login
```

И задеплоить:

```bash Terminal
npx vercel -c --prod
```

В отличие от глитча, здесь надо деплоить каждый раз, когда делаете какие-то изменения локально.

Ссылка на получившийся сайт — вот здесь после слова `Production`. Название сайта можно поменять на [vercel.com](https://vercel.com/) в настройках проекта.
